GLox Language Reference

Introduction

GLox is a bytecode interpreter for the Lox language (from Bob Nystrom's Crafting Interpreters), implemented in Go and extended with Python-inspired features and Raylib graphics bindings.

On top of vanilla Lox, GLox adds: lists, tuples, dictionaries and slicing; exceptions with try/except; module imports with bytecode caching; foreach/range iteration; integer arithmetic and modulus; break/continue; const; implicit variable declaration; string interning; native vector types (vec2/vec3/vec4); a float_array type; a native physics_world simulation; and Raylib bindings for 2D/3D graphics, shaders, textures and batched rendering.

This document is the complete language and library reference. It builds on README.md and the topic docs under docs/ (which live in docs/md/), consolidating and extending them into a single hyperlinked reference.

No semicolons requiredUnlike standard Lox, GLox does not require semicolon line terminators. A newline ends a statement, so ; is optional (both styles are accepted). Most examples in this reference omit them. See Variables.
Reading the signaturesMethod and function signatures are shown in monospace boxes. Arguments in italics describe the expected value type. Unless noted, vec4 colours use 0–255 RGBA components.

Running glox

Build the interpreter and run a script:

// Build
go build -o bin/glox main.go

// Run a script
./bin/glox script.lox

// Start the REPL
./bin/glox --repl

// Set LOX_PATH so module imports resolve
. ./setenv.ps1     // PowerShell
. ./setenv         // bash

The REPL

./bin/glox --repl starts an interactive session. State persists across lines: variables, functions, and classes defined on one line remain in scope for every later line.

> var x = 10
> fun add(a) { return a + x }
> print add(5)
15

Definitions can span multiple lines. When a line leaves brackets open ({ }, ( ), [ ]) or a string unterminated, the prompt changes to ... and input is buffered until the statement is complete:

> fun greet(name) {
...   return "hello, ${name}"
... }
> print greet("world")
hello, world

Pressing Enter on an empty line exits the REPL; pressing it while a multi-line entry is still open (at the ... prompt) cancels that pending entry and returns to the > prompt.

Runtime errors don't end the sessionAn error on one line is reported and the REPL continues — previously defined variables and functions are still available on the next line.

Comments

Line comments start with // and run to the end of the line. There are no block comments.

// This is a comment
a = 1   // trailing comment

Variables & const

GLox supports both explicit (var) and implicit variable declaration. Assigning to a new name creates it; no var is required.

var a = 1     // explicit declaration
b = 2         // implicit — assignment creates the variable
a = a + b     // reassignment

const

const declares a variable that cannot be reassigned. Attempting to reassign a const is a compile-time error.

const PI = 3.14159
const MAX_ENTITIES = 500

Destructuring assignment

Lists and tuples can be unpacked into multiple variables in one statement.

a, b, c = [1, 2, 3]
x, y, z = (10, 20, 30)

Statement terminators

Statements may be terminated with a semicolon ; or simply a newline — both are accepted, so semicolons are optional.

A closing brace } also terminates the statement immediately before it, so braced blocks may be written on a single line: if (x) { print 1 } and if (x) { a() } else { b() } are valid.

Use ; to split statements on one lineBecause a newline is the usual statement terminator, two statements sharing a single line must be separated with a semicolon ; — otherwise the parser cannot tell where one ends and the next begins. This applies inside single-line blocks: while (i < 3) { print i; i = i + 1 } and { const c = 7; print c }. The semicolon is only needed between statements; the last one before } needs none, since the brace terminates it.

Compound assignment

The compound-assignment operators +=, -=, *=, /=, and %= work on variables and object properties. As with the binary operators, / on two integers is integer division and % requires integer operands. Vector properties may also use ++ forms via the property compound path.

a = 5
a += 1        // 6
a -= 2        // 4
a *= 3        // 12
a /= 4        // 3   (integer division)
a %= 2        // 1

p = vec2(1, 2)
p.x += 1      // 2
p.y *= 3      // 6

Conditional expression (ternary)

The C-style ternary operator cond ? a : b evaluates cond and yields a if it is truthy, otherwise b. Only the selected branch is evaluated. It is an expression, so it can appear anywhere a value is expected. It is right-associative, so c1 ? a : c2 ? b : c groups as c1 ? a : (c2 ? b : c).

n = 1
var label = n == 1 ? "item" : "items"   // "item"

// only the chosen branch runs
x = ok ? compute() : fallback()          // fallback() not called when ok is truthy

// right-associative chaining
grade = s >= 90 ? "A" : s >= 80 ? "B" : "C"

Control flow

if / else

if (x > 10) {
    print "big"
} else if (x > 0) {
    print "small"
} else {
    print "non-positive"
}

while

i = 0
while (i < 5) {
    print i
    i = i + 1
}

for

for (var i = 0; i < 10; i = i + 1) {
    if (i == 5) break
    if (i % 2 == 0) continue
    print i
}

foreach

foreach iterates any iterable: lists, tuples, strings, range(), or a class implementing the iterator protocol (see Classes).

foreach (item in [1, 2, 3]) {
    print item
}

foreach (ch in "abc") {
    print ch
}

range

range() returns an efficient native integer iterator. Using foreach (i in range(n)) is significantly faster than an equivalent manual for loop.

foreach (i in range(5))        { print i }   // 0..4
foreach (i in range(2, 8))     { print i }   // 2..7
foreach (i in range(0, 10, 2)) { print i }   // 0,2,4,6,8
Performance tipPrefer foreach (i in range(n)) over a manual counter for loop — range() is a native iterator and avoids per-iteration bytecode overhead.

break / continue

break exits the innermost loop; continue skips to the next iteration. Both work in for, while, and foreach.

Functions

Functions are declared with func (or the alias fun). They are first-class values and support closures.

func add(a, b) {
    return a + b
}

func make_counter() {
    count = 0
    func inc() {
        count = count + 1     // closes over `count`
        return count
    }
    return inc
}

c = make_counter()
print c()   // 1
print c()   // 2

A function with no return returns nil. Functions can be passed as arguments and stored in variables, lists and dicts — see functools for map/filter/reduce.

Default parameters

A parameter may be given a default value with name=expr. The default is used when the caller omits that argument. Defaults are evaluated at call time, once per call — so a fresh value is produced each time (no shared mutable-default trap), the default may be any expression (including a function call), and it may reference earlier parameters.

func greet(name, greeting="hi") { return greeting & ", " & name }
print greet("Sam")            // hi, Sam
print greet("Sam", "yo")      // yo, Sam

func f(n, m=n*2) { return m } // default references an earlier parameter
print f(4)                    // 8

func acc(x, items=[]) { items.append(x); return items }
print acc(1)                  // [ 1 ]   (fresh list every call)
print acc(2)                  // [ 2 ]

All defaulted parameters must come after the non-default (required) ones — a required parameter may not follow a defaulted one.

Variadic parameters (*rest)

A trailing *name parameter collects any surplus positional arguments into a fresh list. It must be the last parameter, and combines with required and default parameters.

func sum(*xs) { var t = 0; foreach (x in xs) { t = t + x } return t }
print sum()                   // 0
print sum(1, 2, 3)            // 6

func lead(a, *rest) { return str(a) & "/" & str(len(rest)) }
print lead("x", 1, 2)         // x/2

func mix(a, b=10, *rest) { ... }   // required, then default, then *rest

Anonymous functions (lambdas)

Omit the name to write a function as an expression. Anonymous functions are ordinary closures — they capture surrounding variables (and this inside a method) just like named ones — and are handy as inline callbacks:

var add = func (a, b) { return a + b }
print add(2, 3)                       // 5

// inline callbacks for higher-order functions
print functools.map([1, 2, 3], func (x) { return x * x })     // [ 1 , 4 , 9 ]
print functools.filter(xs, func (x) { return x > 0 })

// immediately-invoked
print (func () { return 7 })()        // 7

The body is a normal braced block with an explicit return (there is no expression-body shorthand). Since an anonymous function has no name, it cannot refer to itself for recursion — assign it to a variable first and call through that.

func in statement position is a declarationA statement that begins with func is always parsed as a named declaration, so write lambdas where an expression is expected (after =, as a call argument, inside a list/dict, after return). To invoke one immediately, wrap it in parentheses: (func () { … })().

Classes

Classes support single inheritance, an init constructor, instance methods, this, super, and static methods.

class Animal {
    init(name) {
        this.name = name
    }
    speak() {
        return this.name & " makes a sound"
    }
}

class Dog < Animal {          // Dog inherits from Animal
    init(name) {
        super.init(name)       // call parent constructor
    }
    speak() {
        return this.name & " barks"
    }
}

d = Dog("Rex")
print d.speak()   // "Rex barks"

Static methods

Methods declared static are called on the class itself rather than an instance.

class MathHelper {
    static square(x) {
        return x * x
    }
}
print MathHelper.square(5)   // 25

toString magic method

If a class defines toString() returning a string, it is used whenever an instance is printed or passed to str().

class Point {
    init(x, y) { this.x = x; this.y = y }
    toString() { return "Point(" & str(this.x) & ", " & str(this.y) & ")" }
}
print Point(3, 4)   // Point(3, 4)

Iterator protocol

A class becomes foreach-iterable by implementing __iter__ (returning an iterator object) and that iterator implementing __next__, which returns successive values and nil to signal the end.

class Range2 {
    init(n) { this.n = n }
    __iter__() { return Range2Iter(this.n) }
}
class Range2Iter {
    init(n) { this.n = n; this.i = 0 }
    __next__() {
        if (this.i >= this.n) return nil
        v = this.i
        this.i = this.i + 1
        return v
    }
}
foreach (x in Range2(3)) { print x }   // 0, 1, 2

Exceptions

GLox provides try/except exception handling. A built-in Exception base class is available; subclass it for custom exceptions. Handlers bind the caught instance with as.

class MyError < Exception {
    init(msg) {
        super.init(msg)
        this.name = "MyError"
    }
}

try {
    raise MyError("something failed")
} except MyError as e {
    print "caught: " & str(e)
} except Exception as e {
    print "other: " & e.name
}
  • Multiple except clauses handle different exception types; the first matching type wins.
  • try/except blocks may be nested.
  • raise instance throws an exception instance.
  • The runtime raises catchable exceptions such as RunTimeError and EOFError.

Modules & imports

Modules are imported by name. Compiled module bytecode is cached in __loxcache__/<module>.lxc and reused unless the source is newer.

import math
import particle_sys as ps        // alias
import random, colour as clr     // multiple, with alias

from … import

Selectively import names into the current namespace, or import everything with *. This works for both file modules and the native built-in modules.

from math import sin, cos, PI
from colour import *
from gfx import window, batch     // native built-in module

Resolving import foo to a file tries, in order: $LOX_PATH/src/modules/foo.lox (glox's own shared modules); foo.lox in the same directory as the script being run; then foo.lox in any subdirectory of that directory, searched recursively — so a project's own modules can be grouped into subfolders without changing how they're imported. Built-in modules are registered natively and need only be imported, not resolved from disk: sys, os, inspect, colour_utils, gfx (window, batch, texture, shader, camera, …), and physics (physics_world).

Numbers

GLox distinguishes integers from floats. Integer literals have no decimal point; arithmetic between two integers stays integer (including /), while any float operand promotes the result to float.

a = 10
b = 3
print a / b     // 3   (integer division)
print a % b     // 1   (modulus)
print a / 3.0   // 3.3333333333333335  (float)
print int(3.9)  // 3
print float(3)  // 3.0

See the math module for trigonometry, roots, rounding and vector maths, and built-in functions for int(), float(), and rand().

Strings

Strings are immutable and interned (the VM works with integer string IDs internally for fast lookup). They support concatenation, repetition, slicing, membership and iteration.

s = "hello"
print len(s)          // 5
print s & " world"    // concatenation
print "@" * 3         // "@@@"  (repetition, either operand order)
print s[0]            // "h"
print s[1:3]          // "el"   (slice)
print "ell" in s      // true   (substring test)
foreach (c in s) { print c }   // iterate characters

String interpolation

Any ${ expression } inside a string literal is evaluated and spliced into the result. It works in both "…" and '…' literals, and the embedded expression may be arbitrary (arithmetic, calls, indexing, method calls, even nested strings). Each value is stringified via the same path as str() / print, so ints, floats, bools, nil, and class instances (through their toString method) all format correctly.

count = 3;
pct = 42.5;
print "total: ${count} (${pct}%)"   // total: 3 (42.5%)
print "next is ${count + 1}"         // next is 4
print "upper: ${"hi".replace("h", "H")}"  // upper: Hi

Write a literal $ as $$ (so $${x} yields a literal ${x}); a lone $ not followed by { is already literal. Interpolation is pure syntax sugar: "a${x}b" desugars to ("a" & str(x) & "b"), so it composes with everything strings already support.

String methods

s.replace(old, new) → string

Returns a copy of s with all occurrences of old replaced by new.

s.join(list) → string

Joins a list of strings using s as the separator. All list elements must be strings.

print "|".join(["a", "b", "c"])   // "a|b|c"
print "xx".replace("x", "y")      // "yy"

Related built-in functions

  • replace(s, old, new) — free-function form of .replace.
  • format(fmt, args...) — printf-style formatting (wraps Go fmt.Sprintf).
  • str(value) — convert any value to its string form.
  • string.split(s, delim) — split into a list of substrings.
format()format uses Go format verbs: %s (string), %d (integer), %f (float), %v (any). Example: format("%s = %d", name, count). For most cases string interpolation ("${name} = ${count}") is more concise; reach for format when you need precise numeric formatting (width, precision).

Lists

Lists are mutable, ordered, and can hold values of any type.

a = []
b = [1, 2, 3]
b.append(4)          // [1, 2, 3, 4]
x = b[0]             // 1
b[1] = 42            // [1, 42, 3, 4]
c = a & b            // concatenation
if (3 in b) { print "found" }
p, q, r, s = b       // destructuring

Slicing & slice assignment

a = [0, 1, 2, 3, 4]
print a[1:3]     // [1, 2]
print a[:2]      // [0, 1]
print a[2:]      // [2, 3, 4]
print a[:]       // full copy
a[2:5] = [7, 8, 9]   // slice assignment

List methods

list.append(value) → nil

Adds value to the end of the list, in place.

list.remove(index) → nil

Removes the element at the integer index, in place. (Note: removal is by index, not by value.)

list.find(value) → int | nil

Returns the index of the first element equal to value, or nil if not present.

list.length() → int

Returns the number of elements. Equivalent to len(list).

Also availableThe built-in append(list, value) is the free-function form of list.append. Sorting and reversing live in itertools; map/filter/reduce in functools.

Tuples

Tuples are immutable lists, written with parentheses. They support all list operations except those that mutate (no append, no index assignment).

t = (1, 2, 3)
print t[0]        // 1
print len(t)      // 3
print 2 in t      // true
x, y, z = t       // destructuring
foreach (v in t) { print v }
print with commasBecause print takes a single expression, print("a", b, c) actually prints the tuple ("a", b, c), rendered as ( "a" , b , c ). To print values joined without tuple formatting, use string interpolation (print "${a} ${b} ${c}"), concatenate with &, or use format().

Dictionaries

Dictionaries map keys to values.

d = {}
d = {"name": "Rex", "age": 3}
v = d["name"]              // index access
d["age"] = 4               // set
print "name" in d          // membership on keys

Dictionary methods

dict.get(key, default) → value

Returns the value for key, or default if the key is absent (avoids a runtime error on missing keys).

dict.keys() → list

Returns a list of the dictionary's keys.

dict.remove(key) → nil

Removes the entry for key, in place.

d = {"a": 1, "b": 2}
print d.get("c", 0)    // 0
print d.keys()         // list of keys
d.remove("a")

Vectors

vec2, vec3 and vec4 are native fixed-size float vectors. Each is a small heap-allocated object (like lists, dicts and other native types), referenced from its Value the same way any other object is — not unboxed into the value representation the way int/float/bool are. They are used throughout the graphics API (positions, sizes, and RGBA colours as vec4).

Unlike the gfx constructors (window, batch, …), the vector constructors vec2() / vec3() / vec4() are built-in — always available, no import needed — because vectors are core value types rather than graphics objects.

v2 = vec2(1, 2)
v3 = vec3(1, 2, 3)
v4 = vec4(255, 128, 0, 255)   // often an RGBA colour

// Component access & mutation
print v3.x        // 1
v3.y = 5
v3.z += 1

// Vector addition uses ++
sum = vec3(1, 2, 3) ++ vec3(10, 10, 10)   // vec3(11, 12, 13)
ConstructorComponents
vec2(x, y).x .y
vec3(x, y, z).x .y .z
vec4(x, y, z, w).x .y .z .w

For vector length, normalisation, rotation and scaling helpers, see the math module.

In-place addition

Each vector type also has an .add(other) method that mutates the receiver's components directly instead of allocating a new vector — useful in hot loops (e.g. updating an entity's position every frame).

pos = vec2(0, 0)
dp = vec2(1, 1)
pos.add(dp)        // pos becomes vec2(1, 1); dp is unchanged

Caution: unlike ++ (which always allocates a fresh vector), .add() mutates the existing object, so any other reference to that same vector — e.g. a value stored via this.pos = pos in a constructor, when the caller kept its own reference to the argument — is mutated too. Prefer .add() only where the receiver is known not to be aliased elsewhere (a field set from an unshared literal, for example); otherwise use ++.

gfx & physics modules

These create native objects, each documented in its own section. vec2/vec3/vec4 are built-in (core value types, always available), but the graphics constructors and helpers live in the gfx module and the physics constructor in physics — import them before use:

from gfx import *              // window, batch, texture, shader, camera, ...
from physics import physics_world
// or keep them namespaced:
import gfx
win = gfx.window(800, 600)
  • built-in: vec2/vec3/vec4
  • gfx: window, image, texture, render_texture, camera, shader, batch, batch_instanced, float_array, plus the image/colour helpers below
  • physics: physics_world

Colour encoding gfx

encode_rgba(r, g, b) → float

Packs three RGB components (0–255 integers) into a single encoded float, e.g. for storing colours in a float_array. (Despite the name, no alpha channel is encoded.)

decode_rgba(value) → tuple

Unpacks an encoded colour float back into an (r, g, b) tuple.

from gfx import encode_rgba, decode_rgba
c = encode_rgba(255, 128, 0)   // pack RGB into one float
rgb = decode_rgba(c)           // (255, 128, 0)

Image helpers gfx

These fill a float_array that you allocate, then render it with win.draw_array() (or a render texture's draw_array_fast()) or save it with draw_png().

draw_png(filename, float_array, rgb_encoded) → nil

Writes a float_array to a PNG file. Pass rgb_encoded = true when cells hold encode_rgba-packed colours, or false for a grayscale array (values 0–1).

lox_mandel_array(array, height, width, max_iter, xoff, yoff, zoom) → nil

Fills array in place with an RGB-encoded Mandelbrot set.

lox_julia_array(array, width, height, max_iter, cx, cy, scale, xoff, yoff) → nil

Fills array in place with an RGB-encoded Julia set.

from gfx import float_array, lox_mandel_array, draw_png
arr = float_array(800, 600)
lox_mandel_array(arr, 600, 800, 100, 0.0, 0.0, 1.0)   // array, height, width, max_iter, xoff, yoff, zoom
draw_png("mandelbrot.png", arr, true)                 // true: cells are RGB-encoded

window gfx

The window object is the main interface for graphics rendering and input. It wraps a Raylib window and requires a display; graphics scripts should not be run headless.

window(width, height) → window
from gfx import window
win = window(800, 600)
win.init()
while (!win.should_close()) {
    win.begin()
    win.clear(vec4(30, 30, 40, 255))
    win.circle_fill(400, 300, 50, vec4(255, 0, 0, 255))
    win.end()
}
win.close()

batch gfx

A batch accumulates many primitives of one type and renders them all in a single native draw call — dramatically faster than thousands of individual Lox-level draw calls. Ideal for particle systems, large scenes and real-time simulations.

batch(type) → batch

type is one of the win.BATCH_* constants. Supported primitive types: cube, textured cube, sphere, triangle (3 points), and circle (flat 3D disc).

from gfx import window, batch, camera
import random
win = window(800, 600)
win.init()
cam = camera(vec3(0, 50, 100), vec3(0, 0, 0), vec3(0, 1, 0))

cube_batch = batch(win.BATCH_CUBE)
foreach (i in range(1000)) {
    pos   = vec3(random.float(-50, 50), 0, random.float(-50, 50))
    size  = vec3(1, 1, 1)
    color = vec4(255, random.integer(0, 255), 0, 255)
    cube_batch.add(pos, size, color)
}
win.begin_3d(cam)
cube_batch.draw()        // all 1000 cubes in one call
win.end_3d()

batch_instanced gfx

batch_instanced() uses GPU mesh instancing to draw very large numbers of textured cubes (100k+) in a single call. It is specialised for textured-cube instancing where hardware instancing gives the biggest win.

batch_instanced(texture, cube_size, max_instances) → batch_instanced
  • texture — the texture applied to every cube (all instances share it); commonly a render_texture's get_texture().
  • cube_size — edge length (float) of every cube.
  • max_instances — capacity (int); add() beyond this raises a runtime error.

Like batch, entries are added then drawn in one call, but the render path uses hardware instancing. Unlike batch, instances carry only a transform (position + rotation) — the texture and size are fixed at construction. The workflow is: add() every instance, call make_transforms() once to build the transform matrices, then draw(camera) each frame.

from gfx import window, camera, render_texture, batch_instanced
import random
win = window(1200, 800)
win.init()
cam = camera(vec3(0, 20, 60), vec3(0, 0, 0), vec3(0, 1, 0))

rt = render_texture(64, 64)                 // build a texture to sample
rt.clear(vec4(0, 0, 0, 255))
rt.circle_fill(32, 32, 30, vec4(255, 128, 0, 255))

cubes = batch_instanced(rt.get_texture(), 2.0, 100000)
foreach (i in range(50000)) {
    pos = vec3(random.float(-50, 50), random.float(-50, 50), random.float(-50, 50))
    cubes.add(pos, vec3(0, 1, 0), random.float(0, 360))   // position, rotation axis, angle°
}
cubes.make_transforms()                     // build matrices once, after adding

// render loop:
win.begin_3d(cam)
cubes.draw(cam)                             // all 50k cubes in one instanced call
win.end_3d()
MethodDescription
add(position_vec3, axis_vec3, angle)Add an instance at position, rotated angle degrees (float) around axis. Errors if max_instances is exceeded.
make_transforms()Build the instance transform matrices from the added instances. Call once after all add()s and before draw().
draw(camera)Render every instance in a single hardware-instanced call.
count()Number of instances added.
Not customisable with shader()Every batch_instanced renders with the same built-in lighting shader (ambient light only, one shared texture per batch); win.begin_shader_mode() has no effect on it, and instances carry only a transform — no per-instance colour or custom data. There is currently no way to give a batch_instanced a custom look from a .lox script. Use plain batch if you need per-object shader effects.

physics_world physics

physics_world is a native 3D rigid-body simulation for spheres. Each body has a position, velocity, radius and material; step() advances them all — applying gravity, bouncing off the boundary box, and resolving body-to-body collisions — in a single native call. Fixed level geometry (floors, shelves, ramps) can be added as static boxes with add_static_box — optionally tilted, never moving — for dynamic spheres to bounce off.

Creating a world

physics_world(min_vec3, max_vec3, cell_size, gravity_vec3) → physics_world
  • min_vec3 / max_vec3 — opposite corners of the boundary box; bodies bounce off these (each body's radius is accounted for).
  • cell_size — internal collision-grid cell size; should exceed the largest body's diameter.
  • gravity_vec3 — constant acceleration applied to every body each step().

Basic usage

Register a material, add bodies to get their ids, then call step() each frame and read positions back with get_position():

from physics import physics_world
from gfx import window, camera
import random
import colour as col
world = physics_world(vec3(-10, 0, -10), vec3(10, 100, 10), 2.0, vec3(0, -0.01, 0))
mat = world.add_material(0.5, 0.3, 0.99)     // restitution, friction, damping

ids = []
foreach (i in range(20)) {
    pos = vec3(random.float(-5, 5), random.float(5, 20), random.float(-5, 5))
    ids.append(world.add(pos, vec3(0, 0, 0), 0.5, mat))
}

win = window(800, 600)
win.init()
cam = camera(vec3(0, 20, 40), vec3(0, 0, 0), vec3(0, 1, 0))
while (!win.should_close()) {
    world.step(1.0)
    win.begin()
    win.clear(vec4(30, 30, 40, 255))
    win.begin_3d(cam)
    foreach (id in ids) {
        p = world.get_position(id)
        win.sphere(p, 0.5, col.WHITE)
    }
    win.end_3d()
    win.end()
}

Methods

MethodDescription

add(pos_vec3, vel_vec3, radius, material_id)Add a dynamic sphere body; returns a stable integer id used by every other method
add_static_box(pos_vec3, half_extents_vec3, axis_vec3, angle, material_id)Add a fixed box that never moves, optionally tilted angle degrees around axis (0 = axis-aligned) — for floors, shelves, ramps
get_box_transform(id)For a static box body, returns (position, half_extents, axis, angle) — feed straight into win.cube_rotated() so drawing always matches exactly what physics collided against
remove(id)Remove a body from the simulation
get_position(id)Current position of a body as a vec3
add_impulse(id, impulse_vec3)Add an instantaneous velocity change (vel += impulse) to one body

Static boxes from add_static_box are checked directly against every dynamic sphere instead, since a single grid cell can't represent a large platform's true extent — so static geometry can be arbitrarily large regardless of cell_size. Two static bodies never collide with each other.

m = re.search("(?P<user>\w+)@(?P<host>\w+)\.com", "contact: bob@example.com")

Performance notes

GLox runs roughly 1.5–3.6× slower than CPython across the loxcraft benchmark suite. Key implementation notes for writing fast Lox:

  • Prefer foreach (i in range(n)) over manual counter loops — range is a native iterator.
  • Use batch / batch_instanced for large numbers of similar primitives instead of per-object draw calls.
  • Move hot per-object simulation into native types where available (e.g. physics_world) rather than per-object Lox method dispatch.
  • Strings are interned; equality and method/global lookup use integer IDs.

Interpreter-level optimisations include a reduced 32-byte Value struct, slice-indexed globals, string interning, a peephole pass with numeric superinstructions (OP_ADD_II/OP_ADD_FF), inline call frames, and a hoisted dispatch-loop frame context.
